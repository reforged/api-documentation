# Errors

Dans ce guide, nous allons parler de ce qui se passe lorsque quelque chose ne va pas pendant que vous travaillez avec l'API. Les erreurs sont possibles, et la plupart du temps, ce sont les vôtres, pas les nôtres. Examinons les codes d'état et les types d'erreur que vous pouvez rencontrer. {{ className: 'lead' }}

Vous pouvez savoir si votre demande a abouti en vérifiant le code d'état lorsque vous recevez une réponse API. Si la réponse n'aboutit pas, vous pouvez utiliser le type d'erreur et le message d'erreur pour déterminer ce qui n'a pas fonctionné et effectuer un débogage rudimentaire (avant de contacter le support).

<Note>
  Avant de créer une issue sur le repo, sachez que 99% de toutes les erreurs signalées sont en faites des erreurs d'utilisateur.
  Par conséquent, veuillez vérifier soigneusement votre code avant de créer une issue sur le repo de `Ruby`
</Note>

## Status codes

Voici une liste des différentes catégories de codes d'état renvoyés par l'API Ruby. Utilisez-les pour savoir si une requête a abouti.

<Properties>
  <Property name="2xx">
    A 2xx status code indicates a successful response.
  </Property>
  <Property name="4xx">
    A 4xx status code indicates a client error — this means it's a _you_
    problem.
  </Property>
  <Property name="5xx">
    A 5xx status code indicates a server error — you won't be seeing these.
  </Property>
</Properties>

## Error types

<Row>
  <Col>
